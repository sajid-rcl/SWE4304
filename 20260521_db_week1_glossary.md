# Week 01 Glossary

| Term | Definition |
|---|---|
| Entity | A thing of interest that the system stores data about. |
| Attribute | A property that describes an entity. |
| Relationship | An association between two entities. |
| Database | An organised collection of related data. |
| DBMS | Software used to create and manage databases. |
| RDBMS | A DBMS based on the relational model using tables and keys. |
| Primary Key (PK) | An attribute that uniquely identifies a record. |
| Foreign Key (FK) | An attribute that links one table to another. |
| Constraint | A rule that the data must follow. |
| Cardinality | The number of relationships between entities. |
| Optionality | Whether participation in a relationship is mandatory or optional. |
| Schema | The structure/design of the database. |
| Normalisation | The process of reducing redundancy and improving data integrity. |
| Integrity | Ensuring data is accurate and consistent. |
| ER Diagram (ERD) | A visual model of entities and relationships. |
| Conceptual Schema | The high-level logical design of the database. |
| Relational Model | A database model based on tables (relations). |
| Data Independence | The ability to change one level of the database without affecting another. |
| Metadata | Data about the database structure and definitions. |
| Table | A collection of related records organised into rows and columns. |
